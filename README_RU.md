# Scavenger Fabric

Русская инструкция для доработанной версии мода Scavenger.

## Что делает мод

Scavenger запускает спидран на случайный или выбранный блок/предмет. Админ, модератор или хост мира выбирает предмет, усложнение и режим, потом отдельно запускает игру командой `/scavenger startgame`. Побеждает игрок или команда, которые первыми добыли нужный предмет.

Мод работает на Fabric с Fabric API. Есть отдельные jar-файлы для Minecraft `1.21`, `1.21.1`, `1.21.2`, `1.21.3`, `1.21.4`, `1.21.5`, `1.21.6`, `1.21.7`, `1.21.8`, `1.21.9`, `1.21.10` и `1.21.11`.

## Установка

1. Поставь Fabric Loader `0.18.4` или новее.
2. Положи в папку `mods` jar мода, который точно совпадает с твоей версией Minecraft.
3. Поставь зависимости для своей версии Minecraft.
4. На сервере jar мода и зависимости должны быть в `mods` сервера. Игрокам тоже нужен этот мод, чтобы видеть меню, таймер и экран победы.

## Как начать игру

1. Админ, модератор или хост заходит в мир/сервер.
2. Меню Scavenger открывается после входа в мир. При создании мира колесо больше не открывается.
3. Выбери предмет, усложнение и режим: `solo`, `duo` или `trio`.
4. Если нужен командный режим, сначала распредели игроков по командам.
5. Когда все готовы, запускай игру командой `/scavenger startgame`.

Важно: добавление игроков в команду не запускает игру. `/scavenger start`, `/scavenger random` и кнопки меню только подготавливают забег.

## Режимы

- `solo` - каждый игрок сам за себя. Кто первый нашёл предмет, тот победил. После победы всем показывается экран победы, а игроков отключает от мира, чтобы можно было пересоздать мир.
- `duo` - команды до 2 игроков. Если один участник команды нашёл предмет, побеждает вся команда.
- `trio` - команды до 3 игроков. Если один участник команды нашёл предмет, побеждает вся команда.

Игроки без команды в `duo` и `trio` играют как одиночные участники. Мод не даст запустить игру, если команда слишком большая для выбранного режима.

## Команды

Команды управления забегом доступны админам, модераторам с правами или хосту одиночного/LAN мира.

- `/scavenger` - показать краткую помощь.
- `/scavenger help` - показать краткую помощь.
- `/scavenger menu` - открыть меню колес и настроек.
- `/scavenger random` - подготовить случайный предмет и случайное усложнение в режиме `solo`.
- `/scavenger random solo` - подготовить случайный забег в соло.
- `/scavenger random duo` - подготовить случайный забег в дуо.
- `/scavenger random trio` - подготовить случайный забег в трио.
- `/scavenger start <item> <modifier>` - подготовить выбранный предмет и усложнение в режиме `solo`.
- `/scavenger start <item> <modifier> solo` - подготовить выбранный соло-забег.
- `/scavenger start <item> <modifier> duo` - подготовить выбранный дуо-забег.
- `/scavenger start <item> <modifier> trio` - подготовить выбранный трио-забег.
- `/scavenger startgame` - реально запустить подготовленную игру и таймер.
- `/scavenger stop` - остановить и очистить текущий забег.
- `/scavenger clear` - очистить текущий забег.

Примеры:

```mcfunction
/scavenger random duo
/scavenger start minecraft:diamond scavenger:none trio
/scavenger startgame
```

## Команды команд

- `/scavenger addteam <leader> add <player>` - добавить игрока в команду лидера. Пример: `/scavenger addteam minesgames11 add vanya228`.
- `/scavenger addteam <leader> remove <player>` - удалить игрока из команды лидера.
- `/scavenger addteam <leader> list` - показать участников команды.
- `/scavenger addteam <leader> delete` - удалить команду.
- `/scavenger addteam <leader> clear` - то же самое, удалить команду.
- `/scavenger team <leader> add <player>` - короткий вариант команды `addteam`.
- `/scavenger teams` - показать все команды.
- `/scavenger teamlist` - показать все команды.
- `/scavenger teams clear` - очистить все команды.
- `/scavenger unteam <player>` - убрать игрока из любой команды.

## Команды кастомных ников

Кастомный ник используется в сообщениях победы, старта и командах Scavenger. Реальный ник Minecraft не меняется.

- `/scavenger nick` - показать свой ник Scavenger.
- `/scavenger nick <nickname>` - поставить себе кастомный ник.
- `/scavenger nick clear` - очистить свой кастомный ник.
- `/scavenger nick set <player> <nickname>` - админская команда, поставить ник другому игроку.
- `/scavenger nick clearplayer <player>` - админская команда, очистить ник другого игрока.
- `/scavenger nick list` - админская команда, показать все кастомные ники.

Пример:

```mcfunction
/scavenger nick Молния
/scavenger nick set minesgames11 Mines
```

## Одиночный мир, LAN и Radmin VPN

Мод работает не только на выделенном сервере. В одиночном мире, LAN и игре через Radmin VPN хост считается управляющим игроком и может открывать меню, крутить колёса и запускать `/scavenger startgame`.

Для игры с друзьями через LAN/Radmin всем игрокам нужен одинаковый jar Scavenger под их версию Minecraft и подходящие зависимости.

## Усложнения генерации мира

Усложнения, которые меняют тип мира или сид, нельзя честно применить к уже созданному серверному или LAN миру. Поэтому на сервере/LAN мод не запускает такие варианты и показывает сообщение.

В обычном одиночном мире такие усложнения используются через пересоздание мира, потому что тип мира нужно выбрать до старта генерации.

## Что изменено по сравнению со старым модом

- Добавлена полноценная работа на сервере, LAN и одиночном мире.
- Добавлена поддержка Fabric API и отдельных jar под Minecraft `1.21`-`1.21.11`.
- Добавлена серверная синхронизация предмета, усложнения, режима, победителя и таймера.
- Колесо и усложнение больше не открываются при создании мира, меню появляется после входа.
- `/scavenger startgame` теперь отдельно запускает игру. Подготовка забега больше не стартует таймер.
- Добавлены режимы `solo`, `duo` и `trio`.
- Добавлены команды команд: `addteam`, `team`, `teams`, `teamlist`, `unteam`.
- Добавлены кастомные ники через `/scavenger nick`.
- В соло-режиме после победы всем показывается победитель, затем игроков отключает, чтобы пересоздать мир.
- Исправлена проблема с командой, где ID предмета и усложнения склеивались без пробела.
- Для серверов убраны усложнения, которые требуют изменения типа уже созданного мира.
